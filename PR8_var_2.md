# Вариант 2 - **Кейс: «Автоматизация выдачи книг в библиотеке»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации запросов руководство библиотеки **BookFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус запросов (выдавать книги);
* назначать ответственных библиотекарей;
* обрабатывать запросы, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки заявок с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять запросы из файлов;
* изменять статусы запросов;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей запросов.

---

## 🔘 Новые элементы управления

### 📖 **Mark as Issued**

Изменяет статус выбранной заявки:

```json
{ "status": "Issued" }
```

---

### 👤 **Assign Librarian**

Назначает библиотекаря на выбранную заявку.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Mark as Issued** и **Assign Librarian** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Mark as Issued** должна быть неактивной, если статус заявки уже `"Issued"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку запросов.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 запросов** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить библиотекаря для каждой заявки (используя кнопку **Assign Librarian**).

5. Найти все заявки с типом **Borrow**.

6. Для каждой такой заявки:

* выбрать строку в таблице;
* проверить, что кнопка **Mark as Issued** активна (`IsEnabled = true`);
* изменить статус на `"Issued"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает заявки с типом Borrow;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
