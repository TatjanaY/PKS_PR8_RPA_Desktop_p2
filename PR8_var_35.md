# Вариант 35 - **Кейс: «Автоматизация подтверждения поставщиков в системе закупок»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации поставщиков руководство компании **SupplierFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус поставщиков (подтверждать партнёрство);
* назначать ответственных менеджеров;
* обрабатывать поставщиков, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки поставщиков с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять поставщиков из файлов;
* изменять статусы поставщиков;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей поставщиков.

---

## 🔘 Новые элементы управления

### ✔ **Approve Supplier**

Изменяет статус выбранного поставщика:

```json
{ "status": "Approved" }
```

---

### 👤 **Assign Manager**

Назначает менеджера на выбранного поставщика.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Approve Supplier** и **Assign Manager** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Approve Supplier** должна быть неактивной, если статус поставщика уже `"Approved"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку поставщиков.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 поставщиков** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить менеджера для каждого поставщика (используя кнопку **Assign Manager**).

5. Найти всех поставщиков с приоритетом **High**.

6. Для каждого такого поставщика:

* выбрать строку в таблице;
* проверить, что кнопка **Approve Supplier** активна (`IsEnabled = true`);
* изменить статус на `"Approved"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

* корректно реализовано управление `IsEnabled`;

* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает поставщиков с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
