# Вариант 8 - **Кейс: «Автоматизация подтверждения медицинских записей»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации медицинских записей руководство клиники **MedFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус записей (подтверждать приём);
* назначать ответственных медсестёр;
* обрабатывать записи, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки записей с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять записи из файлов;
* изменять статусы записей;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей записей.

---

## 🔘 Новые элементы управления

### ✔ **Confirm Appointment**

Изменяет статус выбранной записи:

```json
{ "status": "Confirmed" }
```

---

### 👤 **Assign Nurse**

Назначает медсестру на выбранную запись.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Confirm Appointment** и **Assign Nurse** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Confirm Appointment** должна быть неактивной, если статус записи уже `"Confirmed"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку записей.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 записей** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить медсестру для каждой записи (используя кнопку **Assign Nurse**).

5. Найти все записи с приоритетом **High**.

6. Для каждой такой записи:

* выбрать строку в таблице;
* проверить, что кнопка **Confirm Appointment** активна (`IsEnabled = true`);
* изменить статус на `"Confirmed"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает записи с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
