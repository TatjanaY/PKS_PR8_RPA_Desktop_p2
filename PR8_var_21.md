# Вариант 21 - **Кейс: «Автоматизация отправки товаров в логистической системе»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации отгрузок руководство компании **ShipFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус отгрузок (отправлять товар);
* назначать ответственных операторов;
* обрабатывать отгрузки, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки отгрузок с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять отгрузки из файлов;
* изменять статусы отгрузок;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей отгрузок.

---

## 🔘 Новые элементы управления

### 🚚 **Ship Item**

Изменяет статус выбранной отгрузки:

```json
{ "status": "Shipped" }
```

---

### 👤 **Assign Operator**

Назначает оператора на выбранную отгрузку.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Ship Item** и **Assign Operator** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Ship Item** должна быть неактивной, если статус отгрузки уже `"Shipped"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку отгрузок.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 отгрузок** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить оператора для каждой отгрузки (используя кнопку **Assign Operator**).

5. Найти все отгрузки с приоритетом **High**.

6. Для каждой такой отгрузки:

* выбрать строку в таблице;
* проверить, что кнопка **Ship Item** активна (`IsEnabled = true`);
* изменить статус на `"Shipped"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает отгрузки с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
