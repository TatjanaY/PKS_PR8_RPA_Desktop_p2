# Вариант 25 - **Кейс: «Автоматизация разрешения инцидентов в системе безопасности»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации инцидентов руководство компании **IncidentFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус инцидентов (разрешать проблемы);
* назначать ответственных операторов;
* обрабатывать инциденты, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки инцидентов с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять инциденты из файлов;
* изменять статусы инцидентов;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей инцидентов.

---

## 🔘 Новые элементы управления

### ✔ **Resolve Incident**

Изменяет статус выбранного инцидента:

```json
{ "status": "Resolved" }
```

---

### 👤 **Assign Operator**

Назначает оператора на выбранный инцидент.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Resolve Incident** и **Assign Operator** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Resolve Incident** должна быть неактивной, если статус инцидента уже `"Resolved"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку инцидентов.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 инцидентов** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить оператора для каждого инцидента (используя кнопку **Assign Operator**).

5. Найти все инциденты с приоритетом **High**.

6. Для каждого такого инцидента:

* выбрать строку в таблице;
* проверить, что кнопка **Resolve Incident** активна (`IsEnabled = true`);
* изменить статус на `"Resolved"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает инциденты с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
