# Вариант 29 - **Кейс: «Автоматизация публикации материалов в контент-системе»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации публикаций руководство компании **ContentFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус материалов (публиковать);
* назначать ответственных редакторов;
* обрабатывать материалы, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки материалов с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять материалы из файлов;
* изменять статусы материалов;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей публикаций.

---

## 🔘 Новые элементы управления

### 📰 **Publish Content**

Изменяет статус выбранной публикации:

```json
{ "status": "Published" }
```

---

### 👤 **Assign Editor**

Назначает редактора на выбранную публикацию.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Publish Content** и **Assign Editor** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Publish Content** должна быть неактивной, если статус публикации уже `"Published"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку публикаций.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 публикаций** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить редактора для каждой публикации (используя кнопку **Assign Editor**).

5. Найти все публикации с приоритетом **High**.

6. Для каждой такой публикации:

* выбрать строку в таблице;
* проверить, что кнопка **Publish Content** активна (`IsEnabled = true`);
* изменить статус на `"Published"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает публикации с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
