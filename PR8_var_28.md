# Вариант 28 - **Кейс: «Автоматизация подтверждения платежей в финансовой системе»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации платежей руководство компании **PayFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус платежей (подтверждать оплату);
* назначать ответственных операторов;
* обрабатывать платежи, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки платежей с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять платежи из файлов;
* изменять статусы платежей;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей платежей.

---

## 🔘 Новые элементы управления

### ✔ **Confirm Payment**

Изменяет статус выбранного платежа:

```json
{ "status": "Confirmed" }
```

---

### 👤 **Assign Operator**

Назначает оператора на выбранный платёж.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Confirm Payment** и **Assign Operator** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Confirm Payment** должна быть неактивной, если статус платежа уже `"Confirmed"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку платежей.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 платежей** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить оператора для каждого платежа (используя кнопку **Assign Operator**).

5. Найти все платежи с приоритетом **High**.

6. Для каждого такого платежа:

* выбрать строку в таблице;
* проверить, что кнопка **Confirm Payment** активна (`IsEnabled = true`);
* изменить статус на `"Confirmed"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает платежи с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
