# Вариант 6 - **Кейс: «Автоматизация подтверждения бронирований переговорных»**

---

## 📖 Ситуация

После внедрения прототипа системы бронирования переговорных руководство компании **RoomFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус бронирований (подтверждать);
* назначать ответственных координаторов;
* обрабатывать бронирования, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки бронирований с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять бронирования из файлов;
* изменять статусы бронирований;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей бронирований.

---

## 🔘 Новые элементы управления

### ✔ **Confirm Booking**

Изменяет статус выбранного бронирования:

```json
{ "status": "Confirmed" }
```

---

### 👤 **Assign Coordinator**

Назначает координатора на выбранное бронирование.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Confirm Booking** и **Assign Coordinator** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Confirm Booking** должна быть неактивной, если статус бронирования уже `"Confirmed"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку бронирований.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 бронирований** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить координатора для каждого бронирования (используя кнопку **Assign Coordinator**).

5. Найти все бронирования с приоритетом **High**.

6. Для каждого такого бронирования:

* выбрать строку в таблице;
* проверить, что кнопка **Confirm Booking** активна (`IsEnabled = true`);
* изменить статус на `"Confirmed"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает бронирования с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
