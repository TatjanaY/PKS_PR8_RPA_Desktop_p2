# Вариант 30 - **Кейс: «Автоматизация завершения задач техобслуживания»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации задач техобслуживания руководство компании **MaintainFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус задач (завершать обслуживание);
* назначать ответственных техников;
* обрабатывать задачи, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки задач с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять задачи из файлов;
* изменять статусы задач;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей задач.

---

## 🔘 Новые элементы управления

### ✔ **Complete Maintenance**

Изменяет статус выбранной задачи:

```json
{ "status": "Completed" }
```

---

### 👤 **Assign Technician**

Назначает техника на выбранную задачу.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Complete Maintenance** и **Assign Technician** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Complete Maintenance** должна быть неактивной, если статус задачи уже `"Completed"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку задач техобслуживания.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 задач** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить техника для каждой задачи (используя кнопку **Assign Technician**).

5. Найти все задачи с приоритетом **High**.

6. Для каждой такой задачи:

* выбрать строку в таблице;
* проверить, что кнопка **Complete Maintenance** активна (`IsEnabled = true`);
* изменить статус на `"Completed"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает задачи с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
