# Вариант 23 - **Кейс: «Автоматизация выполнения заданий печати»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации заданий печати руководство компании **PrintFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус заданий (распечатывать);
* назначать ответственных операторов;
* обрабатывать задания, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки заданий с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять задания из файлов;
* изменять статусы заданий;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей заданий.

---

## 🔘 Новые элементы управления

### 🖨️ **Print Document**

Изменяет статус выбранного задания:

```json
{ "status": "Printed" }
```

---

### 👤 **Assign Operator**

Назначает оператора на выбранное задание.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Print Document** и **Assign Operator** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Print Document** должна быть неактивной, если статус задания уже `"Printed"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку заданий печати.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 заданий** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить оператора для каждого задания (используя кнопку **Assign Operator**).

5. Найти все задания с приоритетом **High**.

6. Для каждого такого задания:

* выбрать строку в таблице;
* проверить, что кнопка **Print Document** активна (`IsEnabled = true`);
* изменить статус на `"Printed"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает задания с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
