# Вариант 26 - **Кейс: «Автоматизация активации подписок в корпоративной системе»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации подписок руководство компании **SubscribeFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус подписок (активировать);
* назначать ответственных менеджеров;
* обрабатывать подписки, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки подписок с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять подписки из файлов;
* изменять статусы подписок;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей подписок.

---

## 🔘 Новые элементы управления

### ✔ **Activate Subscription**

Изменяет статус выбранной подписки:

```json
{ "status": "Active" }
```

---

### 👤 **Assign Manager**

Назначает менеджера на выбранную подписку.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Activate Subscription** и **Assign Manager** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Activate Subscription** должна быть неактивной, если статус подписки уже `"Active"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку подписок.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 подписок** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить менеджера для каждой подписки (используя кнопку **Assign Manager**).

5. Найти все подписки с приоритетом **High**.

6. Для каждой такой подписки:

* выбрать строку в таблице;
* проверить, что кнопка **Activate Subscription** активна (`IsEnabled = true`);
* изменить статус на `"Active"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает подписки с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
