# Вариант 31 - **Кейс: «Автоматизация запуска событий в корпоративной системе»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации событий руководство компании **EventFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус событий (запускать мероприятие);
* назначать ответственных координаторов;
* обрабатывать события, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки событий с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять события из файлов;
* изменять статусы событий;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей событий.

---

## 🔘 Новые элементы управления

### ▶ **Start Event**

Изменяет статус выбранного события:

```json
{ "status": "Started" }
```

---

### 👤 **Assign Coordinator**

Назначает координатора на выбранное событие.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Start Event** и **Assign Coordinator** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Start Event** должна быть неактивной, если статус события уже `"Started"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку событий.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 событий** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить координатора для каждого события (используя кнопку **Assign Coordinator**).

5. Найти все события с приоритетом **High**.

6. Для каждого такого события:

* выбрать строку в таблице;
* проверить, что кнопка **Start Event** активна (`IsEnabled = true`);
* изменить статус на `"Started"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает события с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
