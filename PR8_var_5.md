# Вариант 5 - **Кейс: «Автоматизация активации клиентов в корпоративной системе»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации клиентов руководство компании **ClientFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус клиентов (активировать);
* назначать ответственных менеджеров;
* обрабатывать клиентов, поступающих из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки клиентов с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять клиентов из файлов;
* изменять статусы клиентов;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей клиентов.

---

## 🔘 Новые элементы управления

### ✔ **Activate Client**

Изменяет статус выбранного клиента:

```json
{ "status": "Active" }
```

---

### 👤 **Assign Manager**

Назначает менеджера на выбранного клиента.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Activate Client** и **Assign Manager** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Activate Client** должна быть неактивной, если статус клиента уже `"Active"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку клиентов.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 клиентов** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить менеджера для каждого клиента (используя кнопку **Assign Manager**).

5. Найти всех клиентов с приоритетом **High**.

6. Для каждого такого клиента:

* выбрать строку в таблице;
* проверить, что кнопка **Activate Client** активна (`IsEnabled = true`);
* изменить статус на `"Active"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает клиентов с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
