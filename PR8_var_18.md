# Вариант 18 - **Кейс: «Автоматизация подтверждения возвратов товаров»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации возвратов руководство компании **ReturnFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус возвратов (подтверждать возврат);
* назначать ответственных менеджеров;
* обрабатывать возвраты, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки возвратов с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять возвраты из файлов;
* изменять статусы возвратов;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей возвратов.

---

## 🔘 Новые элементы управления

### ✔ **Approve Return**

Изменяет статус выбранного возврата:

```json
{ "status": "Approved" }
```

---

### 👤 **Assign Manager**

Назначает менеджера на выбранный возврат.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Approve Return** и **Assign Manager** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Approve Return** должна быть неактивной, если статус возврата уже `"Approved"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку возвратов.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 возвратов** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить менеджера для каждого возврата (используя кнопку **Assign Manager**).

5. Найти все возвраты с приоритетом **High**.

6. Для каждого такого возврата:

* выбрать строку в таблице;
* проверить, что кнопка **Approve Return** активна (`IsEnabled = true`);
* изменить статус на `"Approved"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает возвраты с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
