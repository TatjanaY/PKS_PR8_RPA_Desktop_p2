# Вариант 20 - **Кейс: «Автоматизация запуска проектов в корпоративной системе»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации проектов руководство компании **ProjectFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус проектов (запускать выполнение);
* назначать ответственных менеджеров;
* обрабатывать проекты, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки проектов с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять проекты из файлов;
* изменять статусы проектов;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей проектов.

---

## 🔘 Новые элементы управления

### ▶ **Start Project**

Изменяет статус выбранного проекта:

```json
{ "status": "Started" }
```

---

### 👤 **Assign Manager**

Назначает менеджера на выбранный проект.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Start Project** и **Assign Manager** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Start Project** должна быть неактивной, если статус проекта уже `"Started"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку проектов.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 проектов** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить менеджера для каждого проекта (используя кнопку **Assign Manager**).

5. Найти все проекты с приоритетом **High**.

6. Для каждого такого проекта:

* выбрать строку в таблице;
* проверить, что кнопка **Start Project** активна (`IsEnabled = true`);
* изменить статус на `"Started"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает проекты с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
