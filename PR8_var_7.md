# Вариант 7 - **Кейс: «Автоматизация подтверждения складских операций»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации складских операций руководство компании **WarehouseFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус операций (подтверждать выполнение);
* назначать ответственных супервайзеров;
* обрабатывать операции, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки операций с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять операции из файлов;
* изменять статусы операций;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей операций.

---

## 🔘 Новые элементы управления

### ✔ **Approve Operation**

Изменяет статус выбранной операции:

```json
{ "status": "Approved" }
```

---

### 👤 **Assign Supervisor**

Назначает супервайзера на выбранную операцию.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Approve Operation** и **Assign Supervisor** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Approve Operation** должна быть неактивной, если статус операции уже `"Approved"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку операций.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 операций** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить супервайзера для каждой операции (используя кнопку **Assign Supervisor**).

5. Найти все операции с приоритетом **High**.

6. Для каждой такой операции:

* выбрать строку в таблице;
* проверить, что кнопка **Approve Operation** активна (`IsEnabled = true`);
* изменить статус на `"Approved"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает операции с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
