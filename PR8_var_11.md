# Вариант 11 - **Кейс: «Автоматизация отправки доставок в логистической системе»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации доставок руководство компании **DeliveryFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус доставок (отправлять в путь);
* назначать ответственных водителей;
* обрабатывать доставки, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки доставок с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять доставки из файлов;
* изменять статусы доставок;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей доставок.

---

## 🔘 Новые элементы управления

### 🚚 **Dispatch Delivery**

Изменяет статус выбранной доставки:

```json
{ "status": "Dispatched" }
```

---

### 👤 **Assign Driver**

Назначает водителя на выбранную доставку.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Dispatch Delivery** и **Assign Driver** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Dispatch Delivery** должна быть неактивной, если статус доставки уже `"Dispatched"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку доставок.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 доставок** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить водителя для каждой доставки (используя кнопку **Assign Driver**).

5. Найти все доставки с приоритетом **High**.

6. Для каждой такой доставки:

* выбрать строку в таблице;
* проверить, что кнопка **Dispatch Delivery** активна (`IsEnabled = true`);
* изменить статус на `"Dispatched"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает доставки с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
