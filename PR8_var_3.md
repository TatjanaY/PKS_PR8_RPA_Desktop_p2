# Вариант 3 - **Кейс: «Автоматизация обработки заказов в интернет-магазине»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации заказов руководство компании **ShopFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус заказов (подтверждать оплату);
* назначать ответственных менеджеров;
* обрабатывать заказы, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки заказов с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять заказы из файлов;
* изменять статусы заказов;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей заказов.

---

## 🔘 Новые элементы управления

### ✅ **Confirm Order**

Изменяет статус выбранного заказа:

```json
{ "status": "Confirmed" }
```

---

### 👤 **Assign Manager**

Назначает менеджера на выбранный заказ.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Confirm Order** и **Assign Manager** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Confirm Order** должна быть неактивной, если статус заказа уже `"Confirmed"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку заказов.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 заказов** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить менеджера для каждого заказа (используя кнопку **Assign Manager**).

5. Найти все заказы с приоритетом **High**.

6. Для каждой такой заявки:

* выбрать строку в таблице;
* проверить, что кнопка **Confirm Order** активна (`IsEnabled = true`);
* изменить статус на `"Confirmed"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает заказы с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
