# Вариант 17 - **Кейс: «Автоматизация запуска онлайн-курсов»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации курсов руководство учебной платформы **CourseFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус курсов (запускать обучение);
* назначать ответственных преподавателей;
* обрабатывать курсы, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки курсов с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять курсы из файлов;
* изменять статусы курсов;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей курсов.

---

## 🔘 Новые элементы управления

### ▶ **Start Course**

Изменяет статус выбранного курса:

```json
{ "status": "Started" }
```

---

### 👤 **Assign Instructor**

Назначает преподавателя на выбранный курс.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Start Course** и **Assign Instructor** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Start Course** должна быть неактивной, если статус курса уже `"Started"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку курсов.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 курсов** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить преподавателя для каждого курса (используя кнопку **Assign Instructor**).

5. Найти все курсы с приоритетом **High**.

6. Для каждого такого курса:

* выбрать строку в таблице;
* проверить, что кнопка **Start Course** активна (`IsEnabled = true`);
* изменить статус на `"Started"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает курсы с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
