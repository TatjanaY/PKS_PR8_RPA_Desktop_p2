# Вариант 34 - **Кейс: «Автоматизация запуска маркетинговых кампаний»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации кампаний руководство компании **MarketFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус кампаний (запускать рекламу);
* назначать ответственных маркетологов;
* обрабатывать кампании, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки кампаний с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять кампании из файлов;
* изменять статусы кампаний;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей кампаний.

---

## 🔘 Новые элементы управления

### ▶ **Launch Campaign**

Изменяет статус выбранной кампании:

```json
{ "status": "Launched" }
```

---

### 👤 **Assign Marketer**

Назначает маркетолога на выбранную кампанию.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Launch Campaign** и **Assign Marketer** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Launch Campaign** должна быть неактивной, если статус кампании уже `"Launched"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку кампаний.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 кампаний** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить маркетолога для каждой кампании (используя кнопку **Assign Marketer**).

5. Найти все кампании с приоритетом **High**.

6. Для каждой такой кампании:

* выбрать строку в таблице;
* проверить, что кнопка **Launch Campaign** активна (`IsEnabled = true`);
* изменить статус на `"Launched"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает кампании с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
