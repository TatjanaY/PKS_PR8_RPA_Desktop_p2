# Вариант 33 - **Кейс: «Автоматизация подтверждения аренд в системе недвижимости»**

---

## 📖 Ситуация

После внедрения прототипа системы регистрации аренд руководство компании **RentFlow** приняло решение расширить её функциональные возможности.

В новой версии системы сотрудники должны иметь возможность:

* изменять статус аренд (подтверждать договор);
* назначать ответственных менеджеров;
* обрабатывать аренды, поступающие из внешних источников данных.

Дополнительно планируется внедрение автоматизации обработки аренд с использованием технологий **Robotic Process Automation (RPA)**. Роботы должны уметь:

* массово добавлять аренды из файлов;
* изменять статусы аренд;
* выполнять операции через пользовательский интерфейс приложения;
* обрабатывать системные окна и исключительные ситуации.

В рамках данной практической работы необходимо **доработать созданное на предыдущей практической работе приложение** и реализовать расширенный сценарий автоматизации.

---

# **Задание 1. Расширение функциональности приложения**

Необходимо **доработать созданное на предыдущей практической работе приложение**, добавив новые элементы управления и логику взаимодействия с таблицей аренд.

---

## 🔘 Новые элементы управления

### ✔ **Confirm Rental**

Изменяет статус выбранной аренды:

```json
{ "status": "Confirmed" }
```

---

### 👤 **Assign Manager**

Назначает менеджера на выбранную аренду.

---

## ⚙️ Управление активностью элементов (IsEnabled)

Необходимо реализовать корректное управление доступностью кнопок:

* кнопки **Confirm Rental** и **Assign Manager** должны быть **неактивны (`IsEnabled = false`)**, если строка таблицы не выбрана;
* при выборе строки кнопки становятся **активными (`IsEnabled = true`)**;
* после выполнения действия кнопки могут оставаться активными или сбрасываться при снятии выделения;
* кнопка **Confirm Rental** должна быть неактивной, если статус аренды уже `"Confirmed"`.

# **Задание 2. Разработка расширенного RPA-сценария**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, выполняющий обработку аренд.

---

## 🤖 Алгоритм работы робота

1. Запустить приложение.

2. Считать данные из Excel-файла.

3. Последовательно добавить **не менее 10 аренд** в систему.

Для каждой записи:

* заполнить поля формы;
* проверить состояние кнопки **Add** (`IsEnabled`);
* при необходимости скорректировать ввод;
* добавить запись в таблицу.

4. Назначить менеджера для каждой аренды (используя кнопку **Assign Manager**).

5. Найти все аренды с приоритетом **High**.

6. Для каждой такой аренды:

* выбрать строку в таблице;
* проверить, что кнопка **Confirm Rental** активна (`IsEnabled = true`);
* изменить статус на `"Confirmed"`.

# 🔎 Требования к корректности выполнения

Работа считается выполненной корректно, если:

### ✔ Интерфейс

* корректно реализовано управление `IsEnabled`;
* кнопки активируются только при выполнении условий;
* корректно обрабатывается выбор строки таблицы;

### ✔ Автоматизация

* робот корректно читает данные из Excel;
* добавляет все записи;
* находит и обрабатывает аренды с приоритетом High;
* корректно работает с элементами интерфейса;

### ✔ Данные

* статусы обновлены корректно;
* данные не искажены;
